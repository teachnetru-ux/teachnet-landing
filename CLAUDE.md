# CLAUDE.md

Рабочие заметки для Claude Code по этому репозиторию.

## Git-процесс
- Разработку вести в рабочей ветке `claude/...`, указанной в задаче; коммитить и пушить туда.
- **Всегда после пуша открывать pull request из рабочей ветки в `main`.**
  Если открытого PR для этой ветки ещё нет — создать новый; если он уже открыт —
  новые коммиты попадают в него автоматически (второй PR не нужен). Если предыдущий PR
  уже смержен, а на ветке появились новые коммиты — создать новый PR.
- Деплой превью (GitHub Pages) идёт из `main` через `.github/workflows/deploy.yml`,
  поэтому изменения видны на сайте только после мёрджа в `main`.

## Фото и логотипы
- Картинки лежат в `public/images/`. Серые `*.svg` там — заглушки.
- Чтобы поставить реальное фото/логотип, положить файл с тем же именем в растровом
  формате (`.webp` / `.jpg` / `.jpeg` / `.png`). Резолвер в `vite.config.ts` на сборке
  сам заменит ссылку `имя.svg` → `имя.<формат>` (приоритет: webp → jpg → jpeg → png).
  Код менять не нужно. Подробности и таблица имён/размеров — в README, раздел «📸 Фото».
- Не трогать: `logo-dark.svg`, `logo-white.svg`, favicon, og-image — они генерируются
  из `logo-full.svg` командой `npm run assets`.
